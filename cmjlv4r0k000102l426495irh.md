React Hooks have revolutionized the way developers write functional components. Introduced in React 16.8, Hooks allow you to use state, side effects, and other React features without writing class components. They make your code cleaner, more readable, and highly reusable. Over 80% of modern React projects now rely on Hooks for state management, lifecycle handling, and performance optimization.

This guide will walk you through the most important React Hooks, their usage, and practical examples.

### What Are React Hooks?

React Hooks are special **functions** that allow you to use React features such as **state** and **lifecycle methods** inside **functional components**. Before Hooks, you had to use **class components** to manage state or lifecycle events. Now, Hooks let you do the same tasks in a simpler and more readable way.

Hooks make your components:

* **Smaller and cleaner** – you can split logic into reusable pieces.
    
* **Easier to understand** – no need for complex class syntax.
    
* **More flexible** – share behavior between components with custom hooks.
    

Hooks enable you to:

* Manage state in functional components.
    
* Perform side effects like fetching data or updating the DOM.
    
* Share logic across components using custom hooks.
    
* Optimize performance by preventing unnecessary re-renders.

## Types of React Hooks

### <mark>1. State Hooks</mark>

#### `useState`

The `useState` hook lets you add state variables to functional components. It is simple and ideal for managing single values like counters, toggles, and form inputs.

**Syntax:**

```javascript
const [state, setState] = useState(initialValue);
```

* `state` is the current state value.
    
* `setState` is a function used to update the state.
    
* `initialValue` is the starting value.
    

**Example: Counter**

```javascript
import React, { useState } from "react";

function Counter() {
    const [count, setCount] = useState(0);

    return (
        <div>
            <h1>Count: {count}</h1>
            <button onClick={() => setCount(count + 1)}>Increment</button>
            <button onClick={() => setCount(count - 1)}>Decrement</button>
        </div>
    );
}

export default Counter;
```

**Key Points:**

* State persists across re-renders.
    
* Updates using `setState` trigger re-render for the specific component.
    
* Works with any type: numbers, strings, arrays, or objects.
    
* Use functional updates for previous state:
    

```javascript
setCount(prevCount => prevCount + 1);
```

`useReducer` is a **Hook** that helps manage **complex state logic** in functional components. It is especially useful when:

* State has multiple parts or sub-values.
    
* State updates depend on the **previous state**.
    
* You want a **clear and predictable** way to handle state transitions.
    

It works similarly to **Redux**, but it’s built into React for local component state.

```javascript
const [state, dispatch] = useReducer(reducer, initialState);
```

* `state` → The **current state** of the component.
    
* `dispatch` → A function used to **send actions** to the reducer.
    
* `reducer` → A function that determines **how the state changes** based on the action.
    
* `initialState` → The initial value of the state.
    

### How `useReducer` Works

1. You define a **reducer function**.
    
2. When an **action** is dispatched, the reducer updates the state based on the action type.
    

```javascript
import { useReducer } from 'react';

// Reducer function
function reducer(state, action) {
  switch (action.type) {
    case 'increment':
      return { count: state.count + 1 };
    case 'decrement':
      return { count: state.count - 1 };
    case 'reset':
      return { count: 0 };
    default:
      return state;
  }
}

function Counter() {
  const initialState = { count: 0 };
  const [state, dispatch] = useReducer(reducer, initialState);

  return (
    <div>
      <p>Count: {state.count}</p>
      <button onClick={() => dispatch({ type: 'increment' })}>Increase</button>
      <button onClick={() => dispatch({ type: 'decrement' })}>Decrease</button>
      <button onClick={() => dispatch({ type: 'reset' })}>Reset</button>
    </div>
  );
}
```

### When to Use `useReducer`

* State has **multiple related values**.
    
* You want to **centralize state updates** in one place (the reducer).
    
* State transitions are **complex** or depend on the **previous state**.
    

For simple state updates, `useState` is enough. But for **complex scenarios**, `useReducer` keeps your code **organized and predictable**.

### <mark>2. Effect Hooks</mark>

#### `useEffect`

Effect Hooks allow functional components to perform **side effects**, which were previously only possible in class components using lifecycle methods like `componentDidMount`, `componentDidUpdate`, and `componentWillUnmount`.

The most commonly used effect hook is `useEffect`.

**Syntax:**

```javascript
useEffect(() => {
    // Effect logic
    return () => {
        // Cleanup logic
    };
}, [dependencies]);
```

* **Effect logic** → Code that runs after the component renders, like fetching data, updating the DOM, or subscribing to events.
    
* **Cleanup function** → Optional. Runs before re-running the effect or when the component unmounts. Useful for clearing timers, unsubscribing, or cleaning resources.
    
* **Dependencies array** → Controls **when the effect runs**:
    
    * `[]` → Runs only once after the first render (like `componentDidMount`).
        
    * `[state]` → Runs whenever a specific state or prop changes.
        
    * No array → Runs after every render.
        

**Example: Update Document Title**

```javascript
import React, { useState, useEffect } from "react";

function App() {
    const [count, setCount] = useState(0);

    useEffect(() => {
        document.title = `Count: ${count}`;
    }, [count]);

    return <button onClick={() => setCount(count + 1)}>Click {count}</button>;
}

export default App;
```

#### `useLayoutEffect` and `useInsertionEffect`

* `useLayoutEffect`: Runs synchronously after DOM updates, useful for measuring layout before the browser paints.
    
* `useInsertionEffect`: Runs before DOM paint to inject styles, especially useful in server-side rendering scenarios.

### <mark>3. Context Hooks</mark>

`useContext` is a Hook that allows you to access **global data** or shared state in your app **without passing props manually through every component**. This solves the problem of **prop drilling**, where you pass data through multiple layers just to reach a deeply nested component.

### How `useContext` Works

1. **Create a context** using `React.createContext()`.
    
2. **Provide a value** to the context using `<Context.Provider>`.
    
3. **Consume the context** in any component using `useContext`.
    

**Syntax:**

```javascript
const value = useContext(MyContext);
```

* `MyContext` → The context object created by `React.createContext()`.
    
* `value` → The current value from the nearest `<MyContext.Provider>` in the component tree.
    

### Example: Theme Management

```javascript
import React, { createContext, useContext, useState } from "react";

// 1. Create a Context
const ThemeContext = createContext();

// 2. Create a Provider component
function ThemeProvider({ children }) {
    const [theme, setTheme] = useState("light");
    const toggleTheme = () => setTheme(prev => (prev === "light" ? "dark" : "light"));

    return (
        <ThemeContext.Provider value={{ theme, toggleTheme }}>
            {children}
        </ThemeContext.Provider>
    );
}

// 3. Consume context in a nested component
function ThemedComponent() {
    const { theme, toggleTheme } = useContext(ThemeContext);

    return (
        <div
            style={{
                background: theme === "light" ? "#fff" : "#333",
                color: theme === "light" ? "#000" : "#fff",
                padding: "20px"
            }}
        >
            <p>Current Theme: {theme}</p>
            <button onClick={toggleTheme}>Toggle Theme</button>
        </div>
    );
}

// 4. Use Provider at the top level
function App() {
    return (
        <ThemeProvider>
            <ThemedComponent />
        </ThemeProvider>
    );
}

export default App;
```

### Key Points

* **Avoid prop drilling:** You don’t need to pass props through intermediate components.
    
* **Global or shared state:** Useful for themes, user authentication, language settings, or any state that multiple components need.
    
* **Works with functional components** only.

### <mark>4. Performance Hooks</mark>

React provides hooks like `useMemo` and `useCallback` to **optimize performance** by preventing unnecessary calculations or re-renders. These are especially useful in large applications or components that render frequently.

### `useMemo`

**Purpose:**  
`useMemo` **memorizes a computed value** so that it is **recalculated only when its dependencies change**. This avoids expensive computations on every render.

**Syntax:**

```javascript
const memoizedValue = useMemo(() => computeExpensiveValue(a, b), [a, b]);
```

* `computeExpensiveValue(a, b)` → The calculation you want to memoize.
    
* `[a, b]` → Dependency array. The function runs **only when these values change**.
    
* `memoizedValue` → Stores the result of the computation.
    

**Example: Expensive Calculation**

```javascript
import React, { useState, useMemo } from "react";

function ExpensiveCalculation() {
    const [number, setNumber] = useState(0);

    const squaredNum = useMemo(() => {
        console.log("Calculating square...");
        return number * number; // expensive calculation
    }, [number]); // recalculates only when 'number' changes

    return (
        <div>
            <input type="number" value={number} onChange={e => setNumber(+e.target.value)} />
            <p>Square: {squaredNum}</p>
        </div>
    );
}
```

Here, the square calculation only runs when `number` changes, not on every render.

### `useCallback`

**Purpose:**  
`useCallback` **memorizes a function**, preventing it from being **re-created on every render**. This is useful when you pass functions as props to child components to avoid unnecessary re-renders.

**Syntax:**

```javascript
const memoizedCallback = useCallback(() => {
    doSomething();
}, [dependencies]);
```

**Example: Avoiding unnecessary re-renders**

```javascript
import React, { useState, useCallback } from "react";

function Child({ onClick }) {
    console.log("Child rendered");
    return <button onClick={onClick}>Click Me</button>;
}

function Parent() {
    const [count, setCount] = useState(0);

    const handleClick = useCallback(() => {
        console.log("Button clicked");
    }, []); // function stays the same across renders

    return (
        <div>
            <p>Count: {count}</p>
            <button onClick={() => setCount(count + 1)}>Increase Count</button>
            <Child onClick={handleClick} />
        </div>
    );
}
```

* Without `useCallback`, `handleClick` would be re-created on every render, causing `Child` to re-render even if it doesn’t need to.
    
* With `useCallback`, the function reference stays the same unless dependencies change, improving performance.

### <mark>5. Reference Hooks</mark>

`useRef` is a React Hook that allows you to **store a value that stays the same across re-renders** of a component. Unlike `useState`, changing a `ref` does **not cause the component to re-render**.

Think of it like a **box where you can keep something**, and that box stays the same even if the component renders again. You can put anything inside it: a DOM element, a number, a string, or an object.

**Use Cases:**

* Accessing DOM elements.
    
* Storing previous state values.
    
* Managing timers and intervals.
    

**Example: Focus Input**

```javascript
import React, { useRef } from "react";

function App() {
    const inputRef = useRef(null);

    const handleClick = () => inputRef.current.focus();

    return (
        <div>
            <input ref={inputRef} />
            <button onClick={handleClick}>Focus Input</button>
        </div>
    );
}

export default App;
```

### <mark>6. Custom Hooks</mark>

**Custom Hooks** are functions you create yourself that **reuse React logic** across multiple components. They let you **encapsulate behavior** like state management, side effects, or subscriptions, keeping your code **clean and organized**.

Custom hooks always start with the word `use`, e.g., `useWidth`, `useAuth`, `useFetch`.

### Why Use Custom Hooks?

* **Reusability**: You can share logic across different components without repeating code.
    
* **Separation of concerns**: Each hook handles a specific task, making components smaller and easier to read.
    
* **Encapsulation**: Components using the hook don’t need to know the implementation details.
    

### Example: Window Width Tracker

This custom hook tracks the **browser window width** and updates it whenever the window is resized.

```javascript
import { useState, useEffect } from "react";

// Custom hook
function useWidth() {
    const [width, setWidth] = useState(window.innerWidth);

    useEffect(() => {
        const handleResize = () => setWidth(window.innerWidth);

        window.addEventListener("resize", handleResize);

        // Cleanup on unmount
        return () => window.removeEventListener("resize", handleResize);
    }, []);

    return width;
}

// Usage in a component
function App() {
    const width = useWidth();

    return <p>Window width: {width}px</p>;
}
```

## Hooks vs Class Components

| Feature | Class Components | React Hooks |
| --- | --- | --- |
| State Management | `this.state` | `useState`, `useReducer` |
| Lifecycle Handling | Lifecycle methods | `useEffect` |
| Reusability | Hard | Easy via custom hooks |
| Code Complexity | Spread across methods | Small, focused functions |
| Learning Curve | OOP familiarity | Functional mindset |

## Key Takeaways

1. Hooks make functional components powerful and reusable.
    
2. `useState` and `useReducer` handle state effectively.
    
3. `useEffect` replaces lifecycle methods for side effects.
    
4. `useContext` avoids prop drilling for global state.
    
5. `useMemo` and `useCallback` improve performance.
    
6. `useRef` persists values across renders without re-rendering.
    
7. Custom hooks improve code reusability.
    

React Hooks are essential for modern React development. Learning them thoroughly allows you to write **clean, maintainable, and performant code** while fully leveraging functional components.
